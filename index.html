Project Overview

We are optimizing our ANN-driven DC motor speed control system based on a feedforward neural network trained by the resilient backpropagation (trainrp) algorithm. The ANN is trained with simulation data from a highly detailed DC motor model, with electrical and mechanical dynamics represented under different load conditions. Embedded in Simulink, the ANN forms part of a closed-loop system, producing accurate control signals for PWM-based speed control with minimal overshoot and improved stability.

Methodology

  • System Modeling: Develop a Simulink model capturing motor electrical (resistance, inductance, back-EMF) and mechanical (inertia, friction) dynamics with PWM closed-loop control.
  • Data Collection & Training: Collect simulation data (voltage, torque, speed) and train a feedforward ANN offline using resilient backpropagation (trainrp) to minimize MSE.
  • Integration: Embed the trained ANN in Simulink via a MATLAB Function block that retrieves the ANN from the workspace and produces real-time control signals.
  • Analysis: Validate system performance by comparing ANN control with conventional PID through response metrics and error analysis.

Understanding Function Block and Outputs

  • Retrieves ANN: Uses evalin with coder.extrinsic to fetch the pre-trained ANN from MATLAB’s workspace.
  • Input Handling: Accepts a feature vector (fixed parameters, voltage, computed torque) matching training dimensions and transposes it for processing.
  • Prediction & Output: Processes the vector through the feedforward ANN (trained via trainrp) and transposes the output for Simulink compatibility.

Challenges

  • Data Availability: Gathering a comprehensive dataset for varied load conditions remains challenging.
  • Multiple Inputs: Ensuring proper scaling and ordering in the ANN function block is complex.
  • Hardware Integration: Transitioning from Simulink to real hardware requires specialized interfacing.

Project Timeline

  • Early March: Defining and training the ANN.
  • Mid March: Integrating the ANN with our circuit and creating our own dataset.
  • Late March: Refining the framework using our dataset and optimizing performance.
